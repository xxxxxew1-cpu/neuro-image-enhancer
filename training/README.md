# Обучение модели

Модель — крошечный CNN-регрессор (~16 тыс. параметров, ~65 КБ весов). Вход —
изображение 64×64×3, выход — 3 числа в диапазоне [-1, 1] (через `tanh`), которые
отображаются в множительные коэффициенты яркости / контраста / цветности
(`factor = exp(scale · output)`, см. `src/shared/params.ts`).

## Как это работает (self-supervised)

Обучение **не требует размеченного датасета**:

1. Генерируем «хорошо экспонированные» изображения с контролируемой статистикой
   (средняя яркость ~0.4–0.55, умеренные контраст и насыщенность), clip-safe.
2. Применяем **случайную** деградацию (яркость/контраст/цветность) — обратные
   параметры известны по построению.
3. Лосс сквозной: модель предсказывает коррекцию → её применяют тем же
   алгоритмом, что и в браузере → сравнивают с «чистым» оригиналом
   (MSE + лёгкая регуляризация против экстремальных значений).

Так модель учится **оценивать и убирать глобальные отклонения** яркости/контраста/
цветности — это свойство не зависит от содержимого и переносится на реальные фото.

## Запуск

```bash
npm run train                 # 700 шагов, batch 16, вход 64 (по умолчанию)
node training/train.mjs 1500 32 64   # дольше/качественнее
node training/verify.mjs      # проверить, что сохранённая модель загружается
```

Результат сохраняется в `public/models/enhance/{model.json, weights.bin}` —
именно оттуда его подхватывает воркер в браузере. Зависимостей сверх tfjs нет
(чистый JS-CPU backend, веса пишутся своим IO-хендлером — `@tensorflow/tfjs-node`
не нужен).

## Повышение качества на реальных данных

Синтетика даёт рабочую модель, но для максимального качества обучите её на
реальных фото:

1. Соберите корпус **хорошо отретушированных, свободно лицензируемых** снимков
   (например, Unsplash/Pexels по их лицензиям, или собственные).
   ⚠️ Не используйте MIT-Adobe FiveK / датасеты Afifi в продакшене — они только
   для исследований.
2. Замените `genClean()` в `training/dsp.mjs` на загрузку реальных изображений
   (декодируйте в тензоры [n,64,64,3] в [0,1]). Остальной конвейер (деградация +
   сквозной лосс) менять не нужно.
3. Валидируйте на реальных, а не только на синтетических изображениях.

## Дальнейшая оптимизация размера (опционально)

Для ещё меньшего размера сконвертируйте в `tfjs_graph_model` с квантизацией
float16 через `tensorflowjs_converter` (требует Python). Текущая модель и так
~65 КБ, поэтому в рамках бюджета 10 МБ это не обязательно.
